## 2026-04-23 — 工具条语义修正（仅前端，后端无影响）

影响范围：**仅前端 UI 层**。后端 API 契约、数据模型、WS 事件协议**零变更**。

### 变更内容

**旧**（错误理解）：工具条三按钮为三个互斥模式 `create / like / drag`。点赞需要切到 `like` 模式后再点卡片。

**新**（定稿）：
- **工具条只有两个互斥模式**：`pointer`（默认，指针状态）/ `drag`（拖动状态）。
- `➕` 是**一次性动作按钮**，不是模式，任何模式下点一下都会弹出「发表新想法」编辑器。
- **点赞是卡片级交互**：每张卡片右下角的心形图标 + 点赞数**始终显示**；在登录态下，对非本人卡片的心形**任何模式都可点击**触发点赞/取消点赞。

### 为什么后端不需要改

| 维度 | 现状 | 是否满足新需求 |
|---|---|---|
| 点赞端点 | `POST /comments/:id/like` / `DELETE` | ✅ 与模式解耦，原本就是卡片级调用 |
| 幂等语义 | 重复点赞 / 重复取消均 `200` | ✅ 无关模式 |
| 权限校验 | 未登录 `401` / 自赞 `403` | ✅ 前端只是调整了**什么时候发**请求，不是**怎么发** |
| `likedByMe` | 按请求者身份计算 | ✅ 不变 |
| WS `comment.liked` 广播 | 带 `actorId / liked / likeCount` | ✅ 不变 |

### 前端需要做的事

1. [FRONTEND-NOTES.md 第 5 节](./FRONTEND-NOTES.md#5-编辑与工具条交互定稿) 已同步修正，对照新表实现工具条。
2. [FRONTEND-NOTES.md 第 9 节](./FRONTEND-NOTES.md#9-状态存储分工) `useBoardModeStore` 类型改为 `"pointer" | "drag"`，默认 `"pointer"`。
3. 卡片组件内**直接挂心形点击处理**（不再受工具条模式门禁），但要按下表做可用性判断：

   | 状态 | 心形外观 | 点击行为 |
   |---|---|---|
   | 未登录 | 灰度、hover 不高亮 | 弹一次「登录后才能点赞」的 toast；**不**自动开登录 modal |
   | 登录 + 本人卡 | 正常颜色，光标 `default`（非 `pointer`） | 不响应 |
   | 登录 + 他人卡 + 未赞 | 线框 `var(--muted-foreground)` | `POST /comments/:id/like` + 乐观更新 |
   | 登录 + 他人卡 + 已赞 | 填充 `#ef4444` | `DELETE /comments/:id/like` + 乐观更新 |

4. `➕` 按钮的行为：点击立即弹编辑器；提交后卡片落在**当前画板视口中心对应的逻辑坐标 ± 200 随机偏移**（防止多次新建卡片堆在同一像素）。

### 后端需要做的事

**无**。不需要改代码、不需要改文档。本条变更仅作为知会，确认 API 层无歧义即可。

### 联调回归点

- [ ] 切到 `pointer` 模式，点他人卡片心形 → 点赞数 +1，WS 广播到其他客户端。
- [ ] 切到 `drag` 模式，点他人卡片心形 → **仍然**能点赞（不再被模式拦截）。
- [ ] 切到 `drag` 模式，按住自己的卡片拖动 → 正常拖动；松手落盘。
- [ ] 未登录态，点任意心形 → toast「登录后才能点赞」，不发出任何请求。
- [ ] 登录态点自己卡片心形 → UI 无反应，**不发出**请求（前端拦住，不走到后端的 `403`）。
